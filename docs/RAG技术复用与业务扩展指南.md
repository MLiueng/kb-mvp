## 二、典型案例：智能商品推荐系统

以"对话式商品推荐"为例，展示如何基于本项目技术栈构建推荐系统。

### 2.1 数据处理端：商品数据分割与向量化入库

#### 转化思路

RAG 处理的是非结构化长文档（分块）；商品推荐处理的是结构化商品记录（聚合为语义文本）。

```
RAG 数据流:                          推荐数据流:
原始文档 (5000字)                    商品记录 (结构化字段)
    │ 分块 (递归字符切分)                 │ 聚合 (字段拼接为语义文本)
    ▼                                    ▼
多个文本块 (500字/块)                 一个语义文本块 (300字/商品)
    │ 向量化                              │ 向量化
    ▼                                    ▼
N 个向量 + locator                   1 个向量 + 商品属性
```

#### 代码实现

```python
# product_loader.py —— 复用本项目的 Embedding 适配层
from app.embedding import EmbeddingService

embedding_service = EmbeddingService()  # 直接复用多供应商适配层

def split_product(product: dict) -> str:
    """将商品结构化数据聚合为可向量化的语义文本。"""
    parts = [
        f"商品名称：{product['title']}",
        f"品牌：{product['brand']}",
        f"类目：{product['category']}",
    ]
    if product.get('description'):
        parts.append(f"商品描述：{product['description']}")
    attrs = product.get('attributes', {})
    if attrs:
        attr_text = "；".join(f"{k}：{v}" for k, v in attrs.items())
        parts.append(f"商品属性：{attr_text}")
    return "\n".join(parts)

# 批量向量化入库（复用 EmbeddingService）
products = [...]  # 商品列表
texts = [split_product(p) for p in products]
embeddings = embedding_service.embed(texts)  # 同一模型、同一接口
```

#### 数据颗粒度差异

| 维度 | RAG 知识库 | 商品推荐 |
|------|-----------|----------|
| 数据类型 | 非结构化长文本 | 结构化字段 |
| 分块关系 | 1 文档 → N 块 | 1 商品 → 1 块 |
| 分块依据 | 语义边界 + 重叠窗口 | 字段聚合模板 |
| 元数据 | locator（溯源高亮） | product_id + 属性（过滤+回链） |
| 数据量级 | 万级 | 十万~百万级 |
| 标量过滤 | 较少 | 丰富（类目/价格/品牌） |

### 2.2 查询交互端：用户意图识别与向量化

#### 转化思路

RAG 的用户提问即检索意图，直接向量化；商品推荐的用户输入模糊口语化，需 LLM 意图识别。

```
RAG 查询:                           推荐查询:
"退款政策是什么？"                   "5000左右的华为手机，拍照好一点"
    │ 直接向量化                          │ LLM 意图识别
    ▼                                    ▼
查询向量                            结构化需求 (类目/品牌/价格/属性)
                                    + 查询向量 + 标量过滤条件
```

#### 代码实现

```python
# intent_recognition.py —— 复用本项目的 LLM 适配层
from app.llm import LLMService

llm = LLMService()  # 直接复用多供应商适配层

INTENT_PROMPT = """你是电商推荐助手。分析用户输入，提取结构化需求，输出 JSON：
{
  "intent": "purchase|compare|browse",
  "category": "商品类目",
  "brand": "品牌（可为空）",
  "price_range": [min, max],
  "attributes": {"核心属性": "需求"},
  "query_text": "用于语义检索的文本"
}
用户输入：{input}"""

def recognize_intent(user_input: str) -> dict:
    """LLM 意图识别，提取结构化需求。"""
    messages = [
        {"role": "system", "content": INTENT_PROMPT},
        {"role": "user", "content": user_input},
    ]
    # 复用 LLM 适配层（非流式调用）
    result = "".join(llm.chat_stream(messages))
    return parse_json(result)

def query_to_vector(intent: dict):
    """意图 → 查询向量 + 标量过滤条件。"""
    query_vector = embedding_service.embed([intent["query_text"]])[0]
    filter_expr = {}
    if intent.get("category"):
        filter_expr["category"] = intent["category"]
    if intent.get("price_range"):
        filter_expr["price"] = {"$gte": intent["price_range"][0],
                                "$lte": intent["price_range"][1]}
    return query_vector, filter_expr
```

#### 意图解析差异

| 维度 | RAG | 商品推荐 |
|------|-----|----------|
| 输入复杂度 | 低（明确提问） | 高（模糊/口语化） |
| 意图类型 | 单一（知识查询） | 多样（购买/比较/浏览） |
| 意图解析 | 无需（查询即意图） | LLM 结构化提取 |
| 标量过滤 | 无 | 必须（价格/品牌/类目） |
| 上下文 | 通常单轮 | 多轮对话依赖 |

### 2.3 搜索匹配端：语义检索与重排序

#### 转化思路

RAG 检索即排序（相似度 Top-K 直接用）；商品推荐需两阶段——先大范围召回，再多维特征重排。

```
RAG 检索:                           推荐检索:
查询向量 → Top-4 检索 → 直接使用      查询向量 → Top-50 召回 → 重排序 → Top-10
         (相似度排序)                          (语义+商业+用户+业务)
```

#### 代码实现

```python
# recommend_engine.py —— 复用本项目的向量检索 + LLM 生成
from app.vector_store import VectorStore

vector_store = VectorStore()  # 复用向量存储与检索

def search_and_rerank(query_vector, filter_expr, user_profile):
    # 阶段一：语义检索召回 Top-50（复用向量检索算法）
    candidates = vector_store.search(
        query_vector, top_k=50, filter=filter_expr
    )

    # 阶段二：多维特征重排序（RAG 无此环节，推荐特有）
    scored = []
    for product in candidates:
        semantic_score = product["score"]               # 语义相似度
        sales_score = normalize(product["sales"])        # 销量热度
        rating_score = product["rating"] / 5.0           # 质量评分
        preference_score = match_preference(product, user_profile)  # 用户偏好

        # 加权融合
        final_score = (
            0.30 * semantic_score +
            0.25 * sales_score +
            0.15 * rating_score +
            0.30 * preference_score
        )
        scored.append({**product, "final_score": final_score})

    scored.sort(key=lambda x: x["final_score"], reverse=True)
    return scored[:10]

def generate_recommendation(intent, ranked_products):
    """LLM 整合生成个性化推荐（复用 LLM 适配层 + SSE 流式）。"""
    messages = [
        {"role": "system", "content": "你是电商推荐助手，根据用户需求推荐 3-5 件商品..."},
        {"role": "user", "content": f"需求：{intent}\n候选商品：{ranked_products}"},
    ]
    # 复用流式 LLM 调用
    for token in llm.chat_stream(messages):
        yield token
```

#### 排序策略差异

| 维度 | RAG | 商品推荐 |
|------|-----|----------|
| 排序阶段 | 单阶段 | 两阶段（召回→精排） |
| 排序依据 | 语义相似度（单一） | 语义+商业+用户+业务（多维） |
| 是否重排 | 否 | 是（LTR / 加权融合） |
| 个性化 | 无 | 用户画像驱动 |
| 业务规则 | 无 | 库存过滤/促销加权 |

## 三、三大核心差异对比

### 3.1 数据颗粒度

```
RAG:    文档 ──分块──► N个文本块 ──向量化──► N个向量 + locator
推荐:    商品 ──聚合──► 1个语义文本 ──向量化──► 1个向量 + 属性字段
```

| 差异 | RAG | 推荐 | 影响 |
|------|-----|------|------|
| 分块方向 | 拆分（大→小） | 聚合（字段→文本） | 分块函数需替换 |
| 颗粒度 | 多块/文档 | 一块/商品 | 元数据结构不同 |
| 标量过滤 | 少 | 丰富 | 需扩展过滤能力 |

### 3.2 意图解析

```
RAG:    用户提问 ──直接向量化──► 查询向量
推荐:    用户输入 ──LLM意图识别──► 结构化需求 ──向量化──► 查询向量 + 过滤条件
```

| 差异 | RAG | 推荐 | 影响 |
|------|-----|------|------|
| 解析深度 | 无 | LLM 结构化提取 | 新增意图识别模块 |
| 输出 | 查询向量 | 向量 + 过滤条件 | 需标量条件提取 |
| 上下文 | 单轮 | 多轮依赖 | 需对话状态管理 |

### 3.3 排序策略

```
RAG:    检索 Top-K ──相似度排序──► 直接使用
推荐:    检索 Top-50 ──多维特征重排──► Top-10 ──LLM生成──► 推荐
```

| 差异 | RAG | 推荐 | 影响 |
|------|-----|------|------|
| 排序阶段 | 单阶段 | 两阶段 | 需新增重排模块 |
| 特征维度 | 语义单一 | 语义+商业+用户+业务 | 需特征工程 |
| 个性化 | 无 | 千人千面 | 需用户画像 |

## 四、更多业务扩展场景

本项目的"向量化 → 语义检索 → LLM 生成"范式可泛化至多种场景：

| 场景 | 数据端适配 | 查询端适配 | 排序端适配 | 复用度 |
|------|-----------|-----------|-----------|--------|
| **电商商品推荐** | 商品字段聚合 | 意图识别+过滤 | LTR+业务规则 | ★★★★☆ |
| **内容资讯推荐** | 文章分块向量化 | 兴趣关键词提取 | 相似度+热度 | ★★★★★ |
| **人才简历匹配** | 简历分块向量化 | 岗位需求解析 | 匹配度+经验 | ★★★★☆ |
| **法律案例检索** | 案例分块向量化 | 案情描述提取 | 相似度+时效 | ★★★★★ |
| **医疗症状匹配** | 症状库向量化 | 症状描述解析 | 相似度+严重度 | ★★★★☆ |
| **房产智能推荐** | 房源信息聚合 | 需求意图解析 | 匹配度+价格 | ★★★★☆ |
| **企业知识库** | 文档分块向量化 | 直接提问 | 相似度排序 | ★★★★★（本项目） |

**通用规律**：凡是涉及"数据向量化 → 语义检索 → 模型生成"的场景，均可复用本项目技术栈，适配点集中在分块规则、意图解析深度和排序策略。

## 六、总结

本项目的核心技术栈具有高度通用性：

```
┌──────────────────────────────────────────────────────────┐
│                    技术栈复用全景                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   可直接复用 (70%)            需适配扩展 (30%)             │
│   ├── Embedding 适配层         ├── 分块规则 (替换)         │
│   ├── LLM 适配层               ├── 意图识别 (新增)         │
│   ├── 向量检索算法             ├── 重排序模型 (新增)       │
│   ├── SSE 流式输出             ├── 标量过滤 (扩展)         │
│   ├── 配置管理                 └── Prompt 模板 (替换)      │
│   └── 向量存储                                                  │
│                                                          │
│   适配三要素：                                            │
│   1. 数据颗粒度 → 分块规则                                │
│   2. 意图解析   → 意图识别深度                            │
│   3. 排序策略   → 单一相似度 vs 多维重排                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**核心观点**：理解"数据颗粒度、意图解析、排序策略"三大差异，即可在本项目技术栈基础上快速构建垂直业务应用。这种技术栈复用思维，适用于所有"检索 + 生成"场景。
